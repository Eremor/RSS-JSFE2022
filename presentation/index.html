React Native

React Native is a framework for developing cross-platform iOS and Android apps:
  • • appeared in early 2015
  • • built on top of React
  • • does not use WebView and HTML technologies
  • • native components are bound in JS and wrapped in React

Advantages

  1. 1. Cross-platform development
  2. 2. Mastering React
  3. 3. Build time is faster than Android Studio
  4. 4. JavaScript is convenient for transferring data over the network
  5. 5. UI development

What does the application consist of?

  • there are no usual div, span, button, input, etc.
  • no familiar CSS
  • no DOM

What is there?

Components

The application is built from platform components - these are native modules wrapped in React components

  • • There are cross-platform: View, Text, Image, Picker, ...
  • • There are iOS specific ones: TabBarIOS, ActionSheetIOS, ...
  • • There are Android specific ones: BackAndroid, ToolbarAndroid, ...

CSS is not real - it is polyfill

  • • Layout with absolute values, no relative values
  • • There is a limited implementation of flexbox properties for layout
  • • Full support for CSSx will never be, it is not needed
  • • In total, about 70 properties have been implemented, in fact, there are enough of them; there is always SVG just in case.

Navigation

Has a number of problems:

  • • imperative API (methods) for managing
  • • You need to immediately think about the interaction with the Navigator
  • • Animations and gestures are difficult to control
  • • NavigatorBar is completely detached from the general life
  • • After designing several screens, you will start to experience pain

Much of the problem is solved with redux.

Animations

  • • Implemented through a special component Animated
  • • There is <Animated.View>, <Animated.Image>, <Animated.Text>
  • • Work outside of React, directly updating native components

Native modules

React Native implements the basics, but not all. If there is no module:

  • • Look for a component in UIExplorer (React Native demo application)
  • • Find the correct module name in OS terms
  • • Search in the source texts, perhaps it is undocumented (how to make a photo, SVG)
  • • js.coach/react-native - plugins for React Native
  • • Give preference to native implementation over JS

conclusions

  • • Knowledge of React and JavaScript will open the way to the world of mobile development
